# Quiz Add - 새 퀴즈 추가

새로운 퀴즈 문제를 기존 형식에 맞춰 추가합니다.

## 입력 파라미터

- **카테고리**: $1 (첫 번째 인자)
- **난이도**: $2 (두 번째 인자, datasci-trivia 전용)
- 입력 형식: `$ARGUMENTS` = "카테고리 난이도" (예: "ml medium")

### 사용 가능한 카테고리 및 난이도

| 파일 | 카테고리 | 난이도 |
|-----|---------|-------|
| quiz-master | general, science, history, geography | (없음) |
| datasci-trivia | ml, stats, python, data | easy, medium, hard |

## 퀴즈 형식

### quiz-master/questions.js 형식
```javascript
{
  question: "문제 내용?",
  options: ["선택지1", "선택지2", "선택지3", "선택지4"],
  correct: 0,  // 정답 인덱스 (0-3)
  explanation: "해설 내용"
}
```

### datasci-trivia/questions.js 형식
```javascript
{
  question: "문제 내용?",
  options: ["선택지1", "선택지2", "선택지3", "선택지4"],
  answer: 0  // 정답 인덱스 (0-3)
}
```

## 검증 가이드라인 (필수 준수)

### 1. 모호한 표현 금지
다음 표현 사용 시 반드시 기준을 명시해야 합니다:

| 금지 표현 | 수정 방법 |
|----------|----------|
| 가장, 제일 | 시점/기준 명시 (예: "2024년 기준") |
| 최초, 처음 | 한정어 추가 (예: "기록상 최초") |
| 최대, 최소 | 측정 기준 명시 (예: "면적 기준 최대") |
| 최고, 최저 | 시점 명시 (예: "현재 기록 최고") |
| 유일 | 범위 한정 (예: "알려진 유일한") |

### 2. 정답 분포 확인
- 같은 카테고리 내에서 정답 위치(0,1,2,3)가 균등하게 분포되어야 함
- 새 문제 추가 전 기존 정답 분포를 확인할 것

### 3. 선택지 작성 규칙
- 선택지 길이가 비슷해야 함
- 정답이 눈에 띄게 길거나 짧으면 안 됨
- 오답도 그럴듯해야 함 (명백히 틀린 보기 지양)

### 4. 문제 품질 기준
- 명확하고 간결한 문장 사용
- 이중 부정 사용 금지
- 하나의 정답만 존재해야 함
- 전문 용어 사용 시 난이도에 맞게 조절

## 수행 작업

1. $1(카테고리)과 $2(난이도)를 확인합니다.
2. 해당 파일의 기존 문제들을 읽어 형식과 정답 분포를 파악합니다.
3. 사용자에게 새 문제 내용을 요청합니다.
4. 검증 가이드라인에 맞는지 확인합니다:
   - 모호한 표현이 있으면 수정 제안
   - 정답 분포가 불균형하면 다른 위치 권장
5. 검증 통과 후 해당 카테고리에 문제를 추가합니다.

## 출력 형식

```
## 새 퀴즈 추가

**대상 파일**: [quiz-master 또는 datasci-trivia]/questions.js
**카테고리**: $1
**난이도**: $2 (해당 시)

### 현재 정답 분포
| 위치 | 개수 |
|-----|-----|
| 0 | N개 |
| 1 | N개 |
| 2 | N개 |
| 3 | N개 |

**권장 정답 위치**: N번 (가장 적은 위치)

### 새 문제 입력
문제 내용, 선택지 4개, 정답, 해설을 입력해 주세요.

### 검증 결과
- [ ] 모호한 표현 없음
- [ ] 정답 분포 적절
- [ ] 선택지 균형 적절
- [ ] 문제 품질 적합

### 추가 완료
[추가된 문제 내용 표시]
```

## 사용 예시

```
/project:quiz-add general          # quiz-master의 general 카테고리에 추가
/project:quiz-add science          # quiz-master의 science 카테고리에 추가
/project:quiz-add ml easy          # datasci-trivia의 ml/easy에 추가
/project:quiz-add stats hard       # datasci-trivia의 stats/hard에 추가
```

## 주의사항

- quiz-master 카테고리는 난이도 인자가 필요 없음
- datasci-trivia 카테고리는 반드시 난이도를 지정해야 함
- 문제 추가 후 `/project:quiz-validate`로 검증 권장
